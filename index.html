"Most people learn the tools. I learn the problems — then build whatever tool it takes to solve them."

I started programming at 14 — not in a classroom, not for grades. I started because I saw a problem and thought: I can build something that fixes this. That instinct never left.

Today I build at the intersection of artificial intelligence, hardware automation, and real software products. From Python-based computer vision systems to full IoT deployments, from hackathon wins to a live educational platform — everything I build is meant to work in the real world, not just a portfolio.

I founded Innovus Hub, Hyderabad's most active college tech club. I mentor students not by teaching theory but by putting them in front of real problems and watching them build through it. That's how I learned. It's the only way that actually works.

I don't believe in "beginner projects". Every project I start, I finish. Every system I build, I deploy. The bar is always: does it actually work?

"I'm not here to learn how to code. I'm here to build things that matter — and I started at 14."

Most people my age are still figuring out which course to take. I was deploying IoT systems at 14, building live products at 17, and mentoring other developers before most had shipped their first project.

I live at the edges of what's possible with code — AI that thinks, hardware that responds, software that ships. Everything I build has one test: does it work in the real world, for real people?

I founded Innovus Hub because I believe the best way to learn is to build. I run workshops, host coding sprints and push every student to ship something real before the session ends. Twenty people and counting — and every single one has shipped something.

I also teach. Not textbook Python — practical, project-first programming that takes someone from zero to functional in the fastest possible way. Multiple students have gone from never writing code to building functional programs under my guidance.
